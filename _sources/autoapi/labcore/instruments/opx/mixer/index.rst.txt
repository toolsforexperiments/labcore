labcore.instruments.opx.mixer
=============================

.. py:module:: labcore.instruments.opx.mixer

.. autoapi-nested-parse::

   Tools for using (IQ) mixers with the QM OPX.

   Required packages/hardware:
   - QM OPX incl python software
   - SignalHound USB SA124B + driver (comes with qcodes)

.. py:class:: MixerCalibration(lo_frq: float, if_frq: float, analyzer: labcore.instruments.qcodes_drivers.SignalHound.Spike.Spike, qm: qm.QuantumMachine, mixer_name: str, element_name: str, pulse_name: str)

   Class for performing IQ mixer calibration.

   We assume that we control the I and Q with a QM OPX, and monitor the output of the mixer with a
   SignalHound spectrum analyzer.
   Requires that independently a correctly specified configuration for the OPX is available.

   :param lo_frq: LO frequency in Hz
   :param if_frq: IF frequency in Hz (can be positive or negative, depending on the sideband you want)
   :param analyzer: SignalHound qcodes driver instance
   :param qm: Quantum Machine instance (with config applied)
   :param mixer_name: the name of mixer we're tuning, as given in the QM config
   :param element_name: the name of the element thats playing the IQ waveform, as given in the QM config
   :param pulse_name: the name of the (CW) pulse we're playing to tune the mixer, as given in the QM config

   .. py:method:: optimize_lo_leakage(initial_guess: numpy.ndarray = np.array([0.0, 0.0]), ranges: Tuple[float, float] = (0.1, 0.1), cb_options: Optional[Dict[str, Any]] = None)

      Optimize the IQ DC offsets using COBYLA.

      The initial guess and ranges are used to specify the initial simplex
      for the COBYLA algorithm.
      initial guess is the starting point, and initial ranges are the distances
      along the two coordinates for the remaining two vertices of the initial simplex.

      :param initial_guess: x0 of the COBYLA algorithm, an array with two elements, for I and Q offset
      :param ranges: distance for I and Q vectors to complete the initial simplex
      :param cb_options: Options to pass to the `scipy.optimize.minimize(method='COBYLA')`.
                         Will be passed via the `options` dictionary.
